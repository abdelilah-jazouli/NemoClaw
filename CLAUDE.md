## Project Overview

NVIDIA NemoClaw is an open-source reference stack (alpha) for running OpenClaw always-on assistants safely inside OpenShell sandboxes. It combines a TypeScript CLI plugin, a Python blueprint for sandbox orchestration, and a Bash CLI entry point, all wrapped in a Docker-based sandbox with four security layers (network, filesystem, process, inference).

## Build & Development Commands

```bash
# Initial setup
npm install                                    # Root deps + auto-installs prek hooks
cd nemoclaw && npm install && npm run build    # Plugin TypeScript compilation
cd nemoclaw-blueprint && uv sync               # Python blueprint deps

# Plugin development
cd nemoclaw && npm run dev                     # Watch mode (tsc --watch)
cd nemoclaw && npm run build                   # One-time compile

# Linting & formatting
make check                                     # All linters via prek (primary entry point)
make format                                    # Auto-format TS and Python
cd nemoclaw && npm run check                   # Plugin-only lint + typecheck

# Tests
npm test                                       # All test projects (cli + plugin)
npx vitest run --project cli                   # CLI integration tests only
npx vitest run --project plugin                # Plugin unit tests only
npx vitest run test/onboard.test.js            # Single test file
npx vitest run --project e2e-brev              # E2E (requires BREV_API_TOKEN)

# Documentation
make docs                                      # Build Sphinx/MyST docs
make docs-live                                 # Serve with auto-rebuild
```

## Architecture

The repo has three main components:

**`bin/`** — CLI entry point (`nemoclaw.js`, CommonJS). Global commands: `onboard`, `connect`, `status`, `logs`, `start/stop`, `uninstall`. Helper modules in `bin/lib/` handle credentials, inference config, onboarding, platform detection, policies, and registry.

**`nemoclaw/`** — TypeScript plugin (ES2022, bundler module resolution). Registers with OpenClaw SDK as a slash command (`/nemoclaw`) and inference provider. Key modules:
- `src/blueprint/runner.ts` — Orchestrates blueprint lifecycle (plan/apply/status/rollback), emits `PROGRESS:<pct>:<label>` to stdout
- `src/blueprint/state.ts` — Persistent state in `~/.nemoclaw/state/nemoclaw.json`
- `src/blueprint/ssrf.ts` — SSRF validation for endpoints
- `src/commands/slash.ts` — Slash command handler
- `src/onboard/config.ts` — Provider/endpoint configuration (supports openai, anthropic, gemini, nim-local, vllm, ollama, custom)

**`nemoclaw-blueprint/`** — Python blueprint with declarative YAML config. `blueprint.yaml` defines sandbox image, inference profiles, and policy. `policies/` contains network egress rules (deny-all base + presets for npm, pypi, docker, slack, etc.).

## Git Conventions

- **Conventional Commits** enforced by commitlint: `<type>(<scope>): <description>`
- Types: `feat`, `fix`, `docs`, `chore`, `refactor`, `test`, `ci`, `perf`
- **Git hooks** managed by prek (not Husky). `npm install` auto-wires hooks via `prepare` script.
  - pre-commit: formatters, linters, Vitest (plugin)
  - commit-msg: commitlint
  - pre-push: `tsc --noEmit`, Pyright

## Test Structure

Vitest with three projects configured in `vitest.config.ts`:
- **cli** — `test/**/*.test.js` (integration tests: onboarding, security, credentials)
- **plugin** — `nemoclaw/src/**/*.test.ts` (unit tests)
- **e2e-brev** — `test/e2e/brev-e2e.test.js` (opt-in, needs `BREV_API_TOKEN`)

Bash E2E tests: `test/e2e-test.sh`, `test/e2e-gateway-isolation.sh` (run inside sandbox).

## Doc-to-Skills Pipeline

Source of truth is `docs/`. Never edit `.agents/skills/docs/` directly — it's autogenerated:
```bash
python scripts/docs-to-skills.py docs/ .agents/skills/docs/ --prefix nemoclaw
```

## Key Requirements

- Node.js 22+, npm 10+
- Python 3.11+ (blueprint, docs)
- Docker (sandbox image builds)
- uv (Python dependency management)
- hadolint (Dockerfile linting)
